# RMBTI 老板出手人格测试 — Phase 2 开发 Spec

> **仓库**：github.com/NingYuleKK/rmbti (Private)
> **当前版本**：main 分支 commit ac8821b
> **永久地址**：https://rmbti-test-9bcmtdkn.manus.space/src/index.html
> **技术栈**：纯前端 HTML/CSS/JS（无框架），部署于 manus.space

---

## 一、背景

RMBTI 是一个面向语音直播平台（PicoPico）老板用户的人格测试 H5。当前版本已完成核心功能：18 道题目、8 主牌 + 4 副签计分引擎、结果页卡牌展示、逆鳞/镜面标签。现在需要做体验层升级，从"能跑的问卷"变成"有入场感的老板人格牌局"。

## 二、P0 需求（最高优先级）

### P0-1：标题和按钮文案更新

**当前**：标题"直播老板 RMBTI 人格测试"，副标题"测测你是哪种老板人格"，按钮"开始测试"

**改为**：
- 标题：**RMBTI 老板出手人格测试**
- 副标题：**测测你在场子里，是哪一张老板牌**
- 按钮：**开始翻牌**

**改动文件**：`src/data.js` 中的 `config.title`、`config.subtitle`；`src/app.js` 中按钮文案。

### P0-2：封面到测试之间加"入场翻牌过场页"

点击"开始翻牌"后，不直接进入 Q1，而是播放一个 2.5-4 秒的过场动画，然后自动进入第一题。

**过场动画分镜**：
1. 第 1 秒：黑金背景，三张牌从远处浮现（scale 从小到大 + opacity 渐入）
2. 第 2 秒：牌面轻微洗动（左右微晃），金币光点轻轻飞过
3. 第 3 秒：中间一张牌背亮起，文案出现
4. 第 3.5-4 秒：牌翻过去，过渡到 Q1

**过场文案**（随机显示一句）：
- "正在洗牌……"
- "正在读取你的出手气场……"
- "主牌即将显影……"
- "请进入老板席。"

**进入 Q1 时的衔接文案**：
> 第一张牌：你出手后，最想看到什么？

**技术要点**：
- 用 CSS animation + JS setTimeout 实现，不需要额外动画库
- 过场页是一个新的 view state（如 `state.view = "intro"`）
- 过场结束后自动切换到 `state.view = "quiz"`

### P0-3：分享卡片修复

**已知问题**：当前 Canvas API 手绘方案在 manus.space 部署环境下，iOS Safari / 微信内置浏览器仍然生成失败。桌面端正常。

**根因分析**：manus.space 通过 iframe/proxy 提供服务，iOS Safari 对跨域 canvas 操作安全策略极严。即使图片已转 base64、QRCode 已本地化，仍可能被 tainted canvas 策略拦截。

**建议修复方向**：
1. 在真实部署环境（非 iframe proxy）下测试，大概率直接能跑
2. 如果仍有问题，考虑后端生成方案（Node.js canvas 或 Puppeteer 截图）
3. 最低保底方案：iOS 下降级为"截图引导"（提示用户截屏当前结果页）

**当前代码位置**：`src/app.js` 中的 `drawShareCard()` 和 `downloadShareCard()` 函数。

### P0-4：分享卡片内容优化

在分享卡片底部二维码上方，加一句挑逗性文案：

**当前**：扫码提示"扫码测测你是哪种老板"

**改为**：动态文案，根据用户结果生成：
> 我测出来是「{主牌名}·{副签名}」
> {逆鳞文案}

例如：

> 来测测你是哪张老板牌。

### P0-5：生产环境隐藏分数，改为显影度

- 显影度计算规则：`(该牌得分 / 该类别最高可能得分) * 100`，取整，显示为百分比
- 显示格式示例：`主牌显影度：君临 86%`，`副签显影度：卡点 74%`

## 三、P1 需求（第二优先级）

### P1-1：题目页氛围化

**当前**：题目页比较素，普通选择题样式。

**改为三层氛围**：

**第一层 - 背景**：保留黑金暗场，不要纯黑。加很弱的牌框纹理、暗金微光、斜向光束（CSS 渐变 + 伪元素实现）。

**第二层 - 题目区域**：像一张"半透明小牌"。题干在牌面上方，选项像四枚可点击牌签（黑底金边卡片）。

**第三层 - 进度提示**：题号改为"第 X / 18 张牌"格式。

**不同类型题的氛围差异**（通过 CSS class 区分）：
- `primary_regular` / `primary_discriminator`：黑金稳重（默认）
- `secondary_regular`：金色更活一点
- `mirror`：更柔一点，像在照镜子
- `primary_turnoff`：边框微红或暗红提示，像进入"逆鳞"

用户不需要知道分类，但视觉上会有节奏。

### P1-2：选中答案反馈 + 轻音效

**选项点击反馈三段**：
1. **点击瞬间**：选项卡轻微下压（transform: scale(0.97)），金边亮起，一枚小金币光点从选项右侧弹出
2. **确认瞬间**（约 0.3s 后）：选项文字变金色，背景从黑变成暗金渐变，边框有一圈光扫过去
3. **进入下一题**（约 0.8s 后）：当前题卡轻轻向左退场，下一题像翻牌一样从右侧进入

**音效**（4 类，需要准备音频文件）：
- 普通选择：轻金币声"叮"（约 0.3s）
- 进入下一题：轻翻牌声（约 0.5s）
- 完成测试：更厚一点的"牌落下 + 金属回响"（约 1s）
- 结果出现：低频"显影音"，像金色门打开（约 1.5s）

**静音开关**：右上角静音图标，默认开启音效，点击可关闭。状态存 localStorage。

**可选**：移动端轻微震动（navigator.vibrate），普通点击短震一下，结果翻牌稍长。

### P1-3：金币进度条

**当前**：普通进度条。

**改为**：黑金轨道 + 18 个小节点。

**交互**：
- 每答一题，一枚小金币从当前选项飞到进度条对应节点
- 当前节点亮起
- 每完成阶段性节点（第 5/10/15/18 题），节点短暂发光一次

**阶段文案**（显示在进度条下方）：
- 1-5 题：洗牌中
- 6-10 题：主牌显影
- 11-15 题：副签归位
- 16-18 题：等待翻牌

**注意**：克制，不要做"满屏金币乱跳"。一枚金币飞入 + 节点亮起就够了。

### P1-4：结果页翻牌动画

结果计算完成后，不直接显示结果，而是有一个 1.5-2 秒的"翻牌显影"动画：
- 一张牌背从中间出现
- 牌面翻转，显示主牌卡牌图
- 卡牌图下方文字渐入
- 配合"显影音"音效

## 六、计分引擎说明

计分引擎在 `src/engine.js`，是纯函数，不依赖 DOM。核心逻辑：

**主牌（8 种）**：deep / saver / ctrl / loyal / myth / rare / king / clutch
- 通过 `primary_regular`（+3 主分 +1 副分）、`primary_turnoff`（+4）、`primary_discriminator`（+5 主分 +1 副分）题型累积分数
- 最高分者为主牌，平分时用 discriminator 和 turnoff 题的选择作为 tiebreaker

**副签（4 种）**：burst（爆冲）/ steady（常陪）/ heroic（豪侠）/ timing（卡点）
- 通过 `secondary_regular` 题型累积分数（+2 或 +3）
- 最高分者为副签

**镜面标签**：`mirror` 题不计分，只记录用户选择的标签（如"排面欢迎""CP 截图""潜伏位""河"）

**组合名称**：主牌中文名 + "·" + 副签中文名，如"君临·卡点"
