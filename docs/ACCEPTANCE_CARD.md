# RMBTI 验收口径卡

## 核心目标（一句话）

用户 2-3 分钟完成 20 题测试，得到"主牌·副签"结果，觉得"靠这就是我"，想分享。

## 三个必须通过的真实用例

1. **完整流程**：用户从首页点"开始测试" → 答完 20 题 → 看到过渡动画 → 进入结果页看到正确的"主牌·副签"组合 + 判词 + 解读
2. **计分准确**：手动构造一组全选 A 的答案，验证主牌和副签的分数计算是否与 PRD 题库的计分规则一致
3. **分享卡**：结果页点击分享 → 生成包含人格牌图、主牌·副签、判词的分享卡

## 两个不能破坏的旧行为

1. 不适用（新项目）

## 最容易被 agent "做假"的点

- 计分逻辑：选项的 +3/+1/+4/+5 权重是否严格按 PRD 题库实现，近邻人格加分是否遗漏
- 平分处理：主牌平分时是否真的按"判别题得分 → 下头题得分"优先级 tiebreak
- 镜面标签：是否真的只记录标签不参与计分

## 失败时应该怎么表现

- 计分错误：结果页显示的主牌/副签与手动计算不一致
- 流程中断：某一题点击选项后没有进入下一题，或最后一题后没有进入结果页
- 视觉跑偏：结果页风格不是黑金神谕牌方向，变成了彩色卡通或企业风

## Litch 最终验收方式

- 自己完整测一遍，看结果是否"被看懂"
- 检查 2-3 个极端计分组合（全选同一人格、平分场景）
- 结果页截图看视觉质量和分享欲
- 配置文件改一道题，刷新后确认生效（验证配置驱动）

## Phase 拆分建议

### Phase 1：核心流程 + 计分引擎
- 配置化数据结构（JSON）
- 20 题完整流程
- 计分逻辑（主牌 + 副签 + 镜面标签）
- 平分 tiebreak
- 结果页基础版（文字为主）

### Phase 2：视觉 + 分享
- 黑金神谕牌视觉风格
- 卡面图片集成（等美术资源）
- 过渡动画
- 分享卡生成
- 镜面标签润色句
